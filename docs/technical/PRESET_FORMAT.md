# Preset Format

> **For AI agents:** This is the canonical specification for the BSC
> preset JSON format. `schemas/preset.schema.json` is generated from
> this document. Every field listed here is authoritative. Before
> generating, validating, or processing any preset object, read this
> document completely. The hard constraints in `CLAUDE.md` section 4
> summarize the most critical rules; this document provides the full
> specification.

---

## Overview

A BSC preset is a self-contained JSON object that completely specifies
an audio session: its group classification, target frequency bands,
voice architecture, multilingual content, evidence grounding, safety
flags, and operational metadata. A preset does not specify user-defined
parameters (respiration duration, session length) — those are recorded
separately in a session instance (see `docs/technical/SESSION_MODEL.md`).

The format has evolved across versions. This document describes **v0.9.1**,
the current catalog version as of April 2026, which includes extended
metadata fields beyond what Seraphony generates. The distinction between
Seraphony output fields and extended catalog fields is noted throughout.

---

## Top-level structure

```json
{
  "_id": "Group - English Name",
  "header": { ... },
  "voices": [ ... ]
}
```

Three top-level keys. No others are permitted.

---

## `_id`

**Type:** string  
**Required:** yes  
**Format:** `"Group - English Name"` — exactly one space, one hyphen,
one space between the group name and the English name.  
**Case:** Group name is capitalized (`Heal`, `Support`, `Perform`,
`Indulge`, `Transcend`). English name uses title case.

```json
"_id": "Perform - Deep Focus"
"_id": "Heal - Anxiety Reset"
"_id": "Support - Tinnitus Quiet"
```

**Known gap:** `_id` is a human-readable string that may change across
catalog versions. It is not a stable persistent identifier. The ontology
uses a versioned IRI (`https://w3id.org/bsc/preset/heal-anxiety-reset`)
as the stable URI. Future catalog versions should add a `uuid` field
at the top level for stable cross-version identification. When generating
RDF instances from presets, derive the IRI from the UUID if present,
otherwise from a normalized form of `_id`.

---

## `header`

The header contains all metadata about the preset. It has three
categories of fields:

1. **Core fields** — generated by Seraphony, required for every valid
   preset
2. **Extended catalog fields** — added during catalog production,
   required for BioSynCare delivery
3. **Backend-injected fields** — set by the backend at creation time,
   never generated by Seraphony or Claude Code

---

### Core header fields (Seraphony output)

#### `group`
**Type:** string enum  
**Required:** yes  
**Values:** `"Heal"` | `"Support"` | `"Perform"` | `"Indulge"` |
`"Transcend"`  
**Case:** always capitalized exactly as shown.

The group classification governs the evidence tier required, the
language permitted in descriptions, and the recommended session
design character. See `docs/concept/EVIDENCE_FRAMEWORK.md` and
the BSC Reference Agent for full routing logic.

#### `version`
**Type:** string  
**Required:** yes  
**Format:** semantic versioning `X.Y.Z`  
**Current:** `"0.9.1"`

#### `purpose`
**Type:** string  
**Required:** yes  
**Format:** Short English phrase describing functional intent. Not
marketing copy. Usually 4–8 words. No period.

```json
"purpose": "sustain calm durable concentration"
"purpose": "ease the transition into sleep"
"purpose": "support planning and strategic clarity"
```

**Note:** In v0.9.1 the catalog also includes `purposeEng`, `purposeIta`,
`purposePrt`, `purposeEsp` — multilingual expansions of this field.
The monolingual `purpose` field remains required as the primary.

#### `targetBand`
**Type:** array of strings  
**Required:** yes  
**Length:** 1–2 entries. More only with explicit rationale.  
**Order:** primary band first, secondary (if any) second.  
**Values:** must use only the controlled vocabulary below.

Primary bands:
```
delta       0.5–4 Hz
theta       4–8 Hz
alpha       8–13 Hz
smr         12–15 Hz
beta        15–30 Hz
gamma       30–42 Hz
```

Sub-bands:
```
low-delta   0.5–2 Hz
high-delta  2–4 Hz
low-theta   4–6 Hz
high-theta  6–8 Hz
low-alpha   8–10 Hz
high-alpha  10–13 Hz
low-beta    15–20 Hz
mid-beta    20–25 Hz
high-beta   25–30 Hz
```

Single-frequency targets:
```
alpha-10    10 Hz   (canonical calming / meditation target)
gamma-40    40 Hz   (specific research-linked case; modality caution mandatory)
```

**Rules:**
- Values are lowercase, exactly as listed.
- Do not use `"gamma"` when you mean `"gamma-40"`. They are distinct
  concepts with different evidence bases.
- Sub-bands may appear alongside their parent only if there is a
  distinct functional reason for both.

#### `hasBreathGuide`
**Type:** boolean  
**Required:** yes  
**Rule:** `true` if and only if exactly one voice in the `voices`
array has `isOn: true`. `false` if no voice has `isOn: true`.
This field must be set accurately — it controls visual breathing
animation in the UI and determines which session arc logic applies.

#### `voiceTypes`
**Type:** array of strings  
**Required:** yes  
**Values:** unique voice type names in first-appearance order from
the `voices` array.  
**Valid values:** `"Binaural"` | `"Martigli"` | `"Martigli-Binaural"`
| `"Symmetry"`

```json
"voiceTypes": ["Martigli-Binaural", "Symmetry"]
"voiceTypes": ["Binaural", "Symmetry"]
"voiceTypes": ["Martigli", "Binaural", "Symmetry"]
```

#### `med2eng`, `med2ita`, `med2prt`, `med2esp`
**Type:** string  
**Required:** all four, always  
**Format:** Short display name in each language. Title case (EN),
sentence case as appropriate for other languages. Do not use all-caps.
Align semantically across languages — the meaning must be consistent,
not necessarily a word-for-word translation.

```json
"med2eng": "Deep Focus",
"med2ita": "Concentrazione Profonda",
"med2prt": "Foco Profundo",
"med2esp": "Foco Profundo"
```

#### `descEng`, `descIta`, `descPrt`, `descEsp`
**Type:** string  
**Required:** all four, always  
**Length:** ≤ 40 words per language. This is a hard limit enforced
by the UI.  
**Framing:** conservative wellness language. See `docs/concept/SCOPE.md`
for permitted and prohibited constructions. No language may be
stronger than another — if the English description hedges, all four
languages must hedge.

#### `uses` (or `usesEng`, `usesIta`, `usesEsp`, `usesPrt` in v0.9.1)
**Type:** array of strings  
**Required:** yes  
**Format:** lowercase, concise tags. These are matched against user
queries by Seraphony.  
**Length:** 8–15 tags typical.

**Note:** v0.9.1 extended the single `uses` field (English only, as
used in Seraphony's output contract) to four language-specific fields
`usesEng`, `usesIta`, `usesEsp`, `usesPrt`. Both forms exist in the
catalog. New presets should use the four-language form.

#### `techDesc` (and `techDescEng`, `techDescIta`, `techDescPrt`, `techDescEsp`)
**Type:** string  
**Required:** yes (at minimum `techDescEng`)

The technical description field explains the actual design choices and
their evidence basis. It is shown to users who want more depth. Rules:

- Describe actual design choices, not generic category copy.
- Do not fabricate evidence, journals, dates, or numeric outcomes.
- Acknowledge limitations when evidence is mixed, indirect, or emerging.
- Keep wellness framing; do not present BSC as medical care.
- May contain no citations if no directly relevant PUBLIC-SAFE
  citation exists — prefer omission over overreach.
- Do not include effect sizes, percentages, or biomarker numbers
  unless directly supported by a citable reference.
- Conclude with "wellness support only; not a treatment claim" or
  equivalent in each language.

**Note:** v0.9.1 uses four separate fields (`techDescEng` etc.) aligned
with the multilingual desc pattern. Older presets may have a single
`techDesc` field. The RDF pipeline should treat the `techDescEng` field
as the primary if both exist.

### Extended catalog fields (v0.9.1, required for BioSynCare delivery)

These fields are not part of Seraphony's output contract but are
required before a preset enters the BioSynCare catalog.

#### `evidenceTier`
**Type:** string enum  
**Required:** yes (for catalog)  
**Values:** `"speculative"` | `"moderate"` | `"established"`

**Note:** This is the current three-value system used in v0.9.1.
The full six-tier system (`speculative`, `anecdotal`, `preliminary`,
`moderate`, `strong`, `established`) is defined in
`docs/concept/EVIDENCE_FRAMEWORK.md` and will be used in the RDF
ontology. Migration will occur when RDF instances are generated.

#### `cautionTags`
**Type:** array of strings  
**Required:** yes (empty array if none)  
**Valid values:**

```
epilepsy-risk          For any preset using strobing visual stimulation
high-intensity         Strong or overwhelming sensory impact
stimulating            Activating; may not be suitable near sleep
fatigue-sensitive      May increase fatigue in susceptible users
emotionally-sensitive  May surface strong emotional responses
drowsy-use             Produces drowsiness; unsafe while driving
nighttime-only         Should only be used before sleep
driving-unsafe         Do not use while operating vehicles
overload-sensitive     May be too intense for sensory-sensitive users
advanced-use           Not recommended for first-time users
ultra-slow-breathing   Very slow breathing parameters; consult physician
                       if respiratory conditions present
```

#### `headphonesMode`
**Type:** string enum  
**Required:** yes  
**Values:** `"required"` | `"recommended"` | `"optional"`

`"required"` for presets where binaural beat function absolutely
depends on dichotic presentation. `"recommended"` for most presets
where the binaural component significantly enhances the experience
but the session is functional without headphones. `"optional"` for
presets with no binaural component.

#### `sessionStyle`
**Type:** string enum  
**Required:** yes  
**Values:** `"background"` | `"active"` | `"contemplative"` |
`"high-intensity"`

`"background"` — designed to run without active attention; suitable
during stationary work. `"active"` — benefits from or requires
attentional engagement. `"contemplative"` — oriented toward inward
experience; Transcend group typical. `"high-intensity"` — Indulge
group extreme presets.

#### `intensityTier`
**Type:** string enum  
**Required:** yes  
**Values:** `"low"` | `"medium"` | `"high"` | `"extreme"`

#### `advancedUser`
**Type:** boolean  
**Required:** yes  
**Description:** `true` for presets not recommended as first
experiences. Advanced presets may require user self-awareness about
sensitivity. Used to filter the default visible preset set.

#### `primaryTargetHz`
**Type:** number  
**Required:** yes  
**Description:** The specific Hz value of the primary entrainment
target. This is the concrete frequency being driven, not the band
label. Derived from the isochronous Symmetry voice's `nnotes / d`
or the binaural beat `|fl - fr|`.

**Example:** A preset with `targetBand: ["smr", "low-beta"]` and a
14 Hz isochronous Symmetry layer has `primaryTargetHz: 14`.

#### `secondaryTargetHzs`
**Type:** array of numbers  
**Required:** no (omit if no secondary target)  
**Description:** Array of Hz values for secondary entrainment targets.

#### `bloomPolicy`
**Type:** string enum  
**Required:** yes  
**Values:** `"avoided"` | `"intentional-soft"` | `"intentional-strong"`

**Bloom** refers to the audible sweep effect when Symmetry voice notes
fall within the frequency range swept by a Martigli or Martigli-Binaural
voice. Inside the sweep range = bloom; outside = clean.

`"avoided"` — all voice fundamentals are placed outside the MB/Martigli
sweep range, producing a clean separated field. `"intentional-soft"` —
some voices fall inside the sweep, producing a subtle bloom with
each breath cycle. `"intentional-strong"` — significant or
intentional bloom for high-intensity or experiential presets.

#### `launchStatus`
**Type:** string enum  
**Required:** yes  
**Values:** `"core"` | `"advanced"` | `"experimental"`

`"core"` — shown by default in the free tier and in Seraphony
recommendations. `"advanced"` — visible in the full preset browser
but not default-shown. `"experimental"` — available but not
recommended for general use; shown only in Lab Player mode.

#### `evidenceBasis`
**Type:** string (free text summary code)  
**Required:** yes (for catalog)  
**Examples:** `"auditory_plus_breathwork"`, `"direct_auditory_cognition"`,
`"direct_auditory_sleep"`, `"preclinical_plus_early_human_av"`

This field summarizes the primary evidence pathway in a machine-
parseable form. Used for internal evidence auditing.

#### `evidenceRefs`
**Type:** array of strings  
**Required:** yes (empty array if no references)  
**Format:** Reference key matching entries in Appendix A of the
BSC Reference Agent.

```json
"evidenceRefs": ["PLATT_HAMMOND_2025", "INGENDOH_2023", "ZACCARO_2018"]
```

#### `evidenceNotes`
**Type:** string  
**Required:** no (recommended)  
**Description:** Human-readable note on the evidence basis and
any important caveats, for internal review.

---

### Backend-injected fields

These are set by the BioSynCare backend at preset creation time.
Seraphony and Claude Code **must not** generate them.

#### `creator`
**Type:** string  
**Description:** Identifier for the creation agent. Examples:
`"fabbri-gpt5.4-content-refresh-v0.9.1"`, `"renato-fabbri-manual"`,
`"seraphony-v1.0"`.

#### `createdAt`
**Type:** ISO 8601 datetime string  
**Description:** Creation timestamp. Never fabricate.

#### `isShownIndex`
**Type:** integer  
**Description:** Display order in the preset list (0-based). Managed
by the backend catalog tool.

#### `isCreatedByAI`
**Type:** boolean  
**Description:** `true` if the preset was generated by Seraphony or
another AI agent, `false` if hand-designed by Renato Fabbri.

---

### Known header gaps

The following fields do not yet exist but are identified as needed
for research reproducibility and ontology alignment:

**`uuid`** — a stable UUID that survives `_id` changes across catalog
versions. Without this, RDF instance IRIs derived from `_id` become
invalid when presets are renamed. Priority: high.

**`derivedFrom`** — reference to the `_id` (or future `uuid`) of
the preset this was revised from. Enables lineage tracking for
iterative design. Priority: medium.

**`targetUserState`** — the intended incoming user state (e.g.,
`"fatigued"`, `"anxious"`, `"neutral"`, `"mentally-active"`).
Currently only the outgoing target is implied. Matching a user's
incoming state to the appropriate preset is a key future capability.
Priority: medium.

**`structuredContraindications`** — currently `cautionTags` is a flat
list. Structured contraindications would allow statements like
"do not use within 2 hours of sleep" or "not suitable for use while
operating machinery" as machine-readable objects. Priority: low for
current phase, required before clinical partner use.

**`inhaleExhaleRatio`** — the ratio of inhale to exhale duration within
a breathing cycle. Currently fixed at 50/50 sinusoidal across all BSC
sessions, so this field is implicit. If this parameter becomes
user-configurable or preset-specific in a future version, it must be
explicit. Priority: low until the feature changes.

## `voices`

**Type:** array of objects  
**Required:** yes  
**Length:** 1–6 voices. Typical: 2–4. Rich presets: 5–6.

Each voice object specifies one sound layer. The `type` field
determines which other fields are permitted. Voice types are:
`"Binaural"`, `"Martigli"`, `"Martigli-Binaural"`, `"Symmetry"`.

**Breathing constraint:** Across all voices in a preset, at most
one voice may have `isOn: true`. This voice carries the breathing
reference: its oscillation frequency controls the visual breathing
animation and panning sync. If no voice has `isOn: true`, there
is no breathing guidance and `hasBreathGuide` must be `false`.

---

### Voice type: `Binaural`

Two sine tones at slightly different frequencies, presented dichotically
(one per ear). The brain perceives a beat at the difference frequency.
Requires headphones for the binaural effect.

**Required fields:**

| Field | Type | Range | Notes |
|---|---|---|---|
| `type` | `"Binaural"` | — | Exact string |
| `fl` | number | 80–1000 Hz | Left ear carrier. Convention: `fl ≤ fr` |
| `fr` | number | 80–1000 Hz | Right ear carrier |
| `waveformL` | number | `0` | Always numeric 0 (sine). Never a string |
| `waveformR` | number | `0` | Always numeric 0 (sine) |
| `panOsc` | integer | 0–3 | Panning mode (see panning table) |
| `panOscPeriod` | number | 0–600 s | Period for modes 1 and 2 |
| `panOscTrans` | number | 0–60 s | Crossfade duration for mode 1 |

**Optional fields:**

| Field | Type | Default | Notes |
|---|---|---|---|
| `iniVolume` | number or null | 0.18 | 0–1. null = use default. >0.30 requires rationale |

**Forbidden:** all Martigli fields (`mf0`, `ma`, `mp0`, `mp1`, `md`,
`isOn`, `waveformM`), all Symmetry fields (`f0`, `nnotes`, `noctaves`,
`d`, `permfunc`, `waveform`).

**Frequency constraints:**
- Beat frequency `|fl - fr|` must be ≤ 35 Hz
- Exception: `|fl - fr| = 40 Hz` is permitted for intentional
  `gamma-40` designs only (must have `targetBand` containing
  `"gamma-40"`)
- Carriers in the golden zone (150–280 Hz) produce warm, comfortable
  sound. Sedative zone (80–120 Hz): deep, heavy. Alertness zone
  (280–400 Hz): bright, stimulating.

**Panning modes:**

| `panOsc` | `panOscPeriod` | `panOscTrans` | Behavior |
|---|---|---|---|
| 0 | 0 | 0 | Static center (no movement) |
| 1 | > `panOscTrans` | 0–60 | Hold & crossfade: sound sits in one ear, crossfades to other over `panOscTrans` s, every `panOscPeriod` s. Typical: `period=120, trans=20` |
| 2 | > 0 | 0 | Sinusoidal: smooth continuous L↔R with period `panOscPeriod`. Typical: 60–160 s period |
| 3 | 0 | 0 | Martigli-synced: spatial motion follows the Martigli breathing oscillation. Use intentionally; changes binaural to effectively monaural during spatial extremes |

---

### Voice type: `Martigli`

A standalone sine tone whose frequency oscillates in a breathing-like
sinusoidal contour. Used for breathing guidance without binaural effect,
or as a textural tremolo layer when `isOn: false`.

**Required fields:**

| Field | Type | Range | Notes |
|---|---|---|---|
| `type` | `"Martigli"` | — | Exact string |
| `mf0` | number | 80–400 Hz | Center frequency of oscillation |
| `ma` | number | 1–400 Hz | Oscillation amplitude. Sweep = `mf0 ± ma`. Typical: `mf0/2` to `3*mf0/4` for audibility |
| `mp0` | number | 0.1–120 s | Initial breathing cycle duration. Must be ≥ 3 s if `isOn: true` (below 3 s is tremolo, not breathing guidance) |
| `mp1` | number | 0.1–240 s | Final breathing cycle duration after `md` seconds. Default: `mp1 ≈ 2 × mp0` |
| `md` | number | 0.1–3600 s | Transition time from `mp0` to `mp1`. Default: 600 s (10 minutes) |
| `isOn` | boolean | — | `true` = this voice is the breathing reference. At most one voice per preset may have `isOn: true` |
| `waveformM` | number | `0` | Always numeric 0 (sine) |

**Optional fields:**

| Field | Type | Default | Notes |
|---|---|---|---|
| `iniVolume` | number or null | 0.25 | 0–1. null = use default |

**Forbidden:** `fl`, `fr`, `waveformL`, `waveformR`, `panOsc`,
`panOscPeriod`, `panOscTrans`, all Symmetry fields.

**Breathing arc behavior:**
The cycle duration linearly interpolates from `mp0` at t=0 to `mp1`
at t=`md`, then holds `mp1` for the remainder of the session. The
inhale/exhale split is always sinusoidal 50/50. See
`docs/technical/BREATHING_MODEL.md` for full specification.

**Multiple Martigli voices:** If a preset has more than one
Martigli or Martigli-Binaural voice, at most one has `isOn: true`.
The others must have `isOn: false` and should have clearly different
`mp0`/`mp1` values (fast tremolo or ultra-slow ambient sweep).

**Frequency bounds check:**
- `mf0 + ma` should not exceed ~450 Hz
- `mf0 - ma` should not fall below ~50 Hz

---

### Voice type: `Martigli-Binaural`

A hybrid that combines the Martigli breathing oscillation with a
binaural beat. Both carriers sweep together at the Martigli frequency,
maintaining a constant beat frequency throughout. The result is a
"living binaural": the timbre and register change with each breath
cycle while the entrainment frequency remains constant.

This is the most commonly used voice type in the BSC catalog for
breathing-enabled presets.

**Required fields:**

| Field | Type | Range | Notes |
|---|---|---|---|
| `type` | `"Martigli-Binaural"` | — | Exact string |
| `fl` | number | 80–1000 Hz | Left carrier (at rest position) |
| `fr` | number | 80–1000 Hz | Right carrier (at rest position) |
| `ma` | number | 1–400 Hz | Oscillation amplitude for both carriers |
| `mp0` | number | 0.1–120 s | Initial breathing cycle duration |
| `mp1` | number | 0.1–240 s | Final breathing cycle duration |
| `md` | number | 0.1–3600 s | Transition time. Default: 600 s |
| `isOn` | boolean | — | `true` = breathing reference |
| `waveformL` | number | `0` | Always numeric 0 |
| `waveformR` | number | `0` | Always numeric 0 |
| `panOsc` | integer | 0–3 | Same panning modes as Binaural |
| `panOscPeriod` | number | 0–600 s | — |
| `panOscTrans` | number | 0–60 s | — |

**Optional fields:**

| Field | Type | Default | Notes |
|---|---|---|---|
| `iniVolume` | number or null | 0.25 | — |

**Forbidden:** `mf0`, `waveformM`, all Symmetry fields.

**Frequency bounds check:**
- Center = `(fl + fr) / 2`
- `center + ma` should not exceed ~450 Hz
- `center - ma` should not fall below ~50 Hz

**Beat frequency:** `|fl - fr|` is fixed and constant throughout
the session despite the oscillating carriers. This is the entrainment
frequency.

---

### Voice type: `Symmetry`

A sequence of notes evenly distributed in time and in pitch (log scale),
repeated in succession with or without permutation. This is the BSC
implementation of sonic symmetry / permuted pitch sequences.

When `noctaves = 0` all notes collapse to the same pitch, producing
a traditional **isochronous (isochronic)** pulse train. This is the
primary mechanism for cortical frequency following at specific Hz values.

**Required fields:**

| Field | Type | Range | Notes |
|---|---|---|---|
| `type` | `"Symmetry"` | — | Exact string |
| `f0` | number | ≥ 80 Hz | Frequency of the lowest note in the sequence |
| `nnotes` | integer | ≥ 2 | Number of notes per cycle |
| `noctaves` | number | ≥ 0 | Pitch span in octaves. 0 = isochronic mode. Highest note = `f0 × 2^((nnotes-1) × noctaves / nnotes)` |
| `d` | number | > 0 s | Cycle duration. The onset interval is `noteSep = d / nnotes` |
| `permfunc` | integer | 0–4 | Permutation function (see table below) |
| `waveform` | number | `0` | Always numeric 0 (sine) |

**Optional fields:**

| Field | Type | Default | Notes |
|---|---|---|---|
| `iniVolume` | number or null | 0.13 | — |

**Forbidden:** all Binaural and Martigli fields including `fl`, `fr`,
`mf0`, `ma`, `mp0`, `mp1`, `md`, `isOn`, `waveformL`, `waveformR`,
`waveformM`, `panOsc`, `panOscPeriod`, `panOscTrans`.

**Timing model (use this, not the deprecated model):**
- Onset interval: `noteSep = d / nnotes`
- Note/pulse rate: `nnotes / d` Hz
- Maximum supported rate: **50 Hz** (`noteSep` ≥ 20 ms)
- DO NOT use the deprecated `d / (N × 2)` abstraction

**Isochronic targeting (noctaves = 0):**
```
Target 10 Hz: nnotes=10, d=1.0  → 10/1.0 = 10 Hz ✓
Target 10 Hz: nnotes=20, d=2.0  → 20/2.0 = 10 Hz ✓
Target 40 Hz: nnotes=8,  d=0.2  → 8/0.2  = 40 Hz ✓
Target 40 Hz: nnotes=20, d=0.5  → 20/0.5 = 40 Hz ✓
INVALID:       nnotes=3,  d=0.05 → 3/0.05 = 60 Hz ✗ (exceeds 50 Hz max)
```

**Permutation functions:**

| `permfunc` | Name | Character | Cognitive engagement |
|---|---|---|---|
| 0 | shuffle | Random reorder each cycle | Highest — maximum unpredictability |
| 1 | rotateForward | Last note becomes first | Low — systematic forward drift |
| 2 | rotateBackward | First note becomes last | Low — systematic backward drift |
| 3 | reverse | Alternating mirrored passes | Moderate |
| 4 | identity | Pure repetition | Lowest — habituation risk for long sessions |

**Symmetry design roles:**

| Role | `d` | `nnotes` | `noctaves` | `permfunc` | Character |
|---|---|---|---|---|---|
| Isochronic | any | any | 0 | any | Traditional pulsed tone; direct entrainment |
| Rhythmic | 0.2–10 s | 2–5 | varies | 4 or 1/2 | Rhythmic pulse with pitch variation |
| Ambient/Sparse | 30–120 s | 4–8 | 1–3 | 0 | Slow evolving background |
| Textural/Dense | < 1 s | 8–20 | 0.2–1 | 0 | Shimmering wash |
| Melodic | 20–60 s | 5–7 | 1–3 | 0 | Contemplative sequences |

**Highest note check:** `f0 × 2^((nnotes-1) × noctaves / nnotes)`
should generally not exceed ~800 Hz without explicit rationale.

## Global constraints across all voice types

### One breathing reference maximum
Across all voices: at most one may have `isOn: true`.

### Volume limits
```
Default by type:
  Martigli / Martigli-Binaural: 0.25
  Binaural:                     0.18
  Symmetry:                     0.13

Hard limits:
  iniVolume > 0.30  → requires explicit rationale in notes
  iniVolume = 1.0   → invalid; do not generate
  iniVolume = null  → valid; engine uses type default
```

### Waveform fields are always numeric zero
`waveformL`, `waveformR`, `waveformM`, `waveform` must always be the
integer or float `0`, never the string `"sine"` or `"0"`.

### Voice count
2–4 voices is standard. 2–3 is minimal (appropriate for Heal/Support
with subtle design intent). 5–6 is rich (appropriate for Indulge or
complex multi-layer Transcend designs). Fewer than 2 is not valid.

## Complete example: v0.9.1 catalog preset

```json
{
  "_id": "Perform - Deep Focus",
  "header": {
    "group": "Perform",
    "version": "0.9.1",
    "purpose": "sustain calm durable concentration",
    "targetBand": ["smr", "low-beta"],
    "hasBreathGuide": false,
    "voiceTypes": ["Binaural", "Symmetry"],
```

## Versioning

The preset catalog is versioned as a whole (`v0.9.1`). Individual
presets carry their own `version` field which should match the catalog
version at time of creation. When a preset is revised, its `version`
is incremented independently of the catalog version.

The `derivedFrom` field (pending, see Known Gaps above) will capture
revision lineage when implemented. Until then, catalog notes in the
backend tool track revisions manually.

## Relationship to the RDF ontology

Each preset in `src/data/presets/` will have a corresponding RDF
instance in `ontology/instances/presets/`. The mapping rules:

- `_id` or `uuid` → IRI at `https://w3id.org/bsc/preset/{normalized-id}`
- `header.group` → `sstim:inGroup` → group individual in `sstim-vocab.ttl`
- `header.targetBand` → `sstim:targetsFrequencyBand` → band concepts
- `header.evidenceTier` → `sstim:evidenceTier` → tier individual
- `header.cautionTags` → `sstim:hasCautionTag` → caution tag individuals
- Voice array → `sstim:composedOf` → voice component individuals

The RDF instances are validated against `ontology/sstim-shapes.ttl`
before the export pipeline generates `dist/presets.json` for
BioSynCare consumption. No preset that fails SHACL validation is
exported.
